InboxBridge

Microsoft OAuth helper

Start the Outlook.com consent flow for a configured source account, then finish the exchange in the callback page. With secure token storage enabled, InboxBridge keeps the refresh token encrypted in PostgreSQL and renews access automatically.

Preview raw JSON

What this page expects

  • Your source already exists in .env with AUTH_METHOD=OAUTH2.
  • Microsoft app settings use the redirect URI https://<admin-ui-host>/api/microsoft-oauth/callback, or whatever you explicitly configured in MICROSOFT_REDIRECT_URI.
  • The source ID here matches the configured bridge source exactly.

After consent

You will land on a callback page that can exchange the code for you. If encrypted token storage is configured, nothing needs to be copied back into the env file.
